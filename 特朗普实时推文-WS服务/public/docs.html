1. 端点 / 鉴权

WS 端点ws://<host>:8787/ws（上 https 反代后用 wss://）
HTTP 基址http(s)://<host>:8787
鉴权方式 AURL 参数 ?token=ttws_live_xxx（浏览器 / 简单 SDK 推荐）
鉴权方式 BHTTP 头 Authorization: Bearer ttws_live_xxx（服务端调用推荐）
鉴权失败HTTP 401，连接立即断
过期 / 吊销下次心跳时强断，close code 4003
API Key 在 /admin 后台创建。可选有效期：1 小时 / 1 天 / 7 天 / 30 天 / 1 年 / 永久 / 自定义。

2. 连接生命周期

client                        server
  | --- HTTP Upgrade ----->     |
  | <--- 101 Switching  ---  |
  | <--- hello              ---|   服务端主动发的第一帧
  | --- subscribe (opt) --->   |
  | <--- subscribed         ---|
  | <--- tweet (real-time) ----|   每条新帖一帧
  | <--- ping (every 25s) -----|   WS 协议层 ping
  | ---  pong  ------------>   |   ws 库自动回 pong
  | --- ping (JSON, opt) -->   |   应用层心跳
  | <--- pong (JSON)        ---|

3. 客户端 → 服务端 消息

type字段说明
ping—应用层心跳；服务端回 pong
subscribesource: "all" | "rss" | "truth"过滤数据源（默认 all）
historylimit?: 1..200 (默认 20)请求最近 N 条历史

4. 服务端 → 客户端 消息

type关键字段说明
hellots, server_version, key连接建立第一帧；key.id / prefix / expires_at
subscribedsource订阅成功确认
tweetdata: Tweet实时新帖
historydata: Tweet[]历史回放
pongts心跳应答
errormessage错误（不会自动断）

6. Python 客户端

import asyncio, json, websockets

URL = "ws://YOUR-HOST:8787/ws?token=ttws_live_xxxxxxxxxxxxxxxxx"

async def main():
    async for ws in websockets.connect(URL, ping_interval=20):
        try:
            await ws.send(json.dumps({"type": "subscribe", "source": "all"}))
            await ws.send(json.dumps({"type": "history", "limit": 20}))
            async for raw in ws:
                msg = json.loads(raw)
                if msg["type"] == "tweet":
                    t = msg["data"]
                    print(f"[{t['source']}] {t['pubDate']} | {t['title'][:80]}")
        except websockets.ConnectionClosed:
            continue

asyncio.run(main())

7. Node.js 客户端

const WebSocket = require('ws'); // npm i ws
const URL = 'ws://YOUR-HOST:8787/ws?token=ttws_live_xxxxxxxxxxxxxxxxx';

function connect() {
  const ws = new WebSocket(URL);
  ws.on('open', () => {
    ws.send(JSON.stringify({ type: 'subscribe', source: 'all' }));
    ws.send(JSON.stringify({ type: 'history', limit: 20 }));
  });
  ws.on('message', raw => {
    const m = JSON.parse(raw);
    if (m.type === 'tweet') console.log(`[${m.data.source}] ${m.data.title.slice(0, 80)}`);
  });
  ws.on('close', () => setTimeout(connect, 1500));
  ws.on('error', e => console.error(e.message));
}
connect();

9. 错误码 / 重连

状况HTTP / WS Close说明
未带 tokenHTTP 401 (missing_token)补 token 后重连
token 格式不对401 (bad_format)必须 ttws_live_ 开头
token 不存在401 (not_found)已被删除
token 已撤销401 / Close 4003 (revoked)找管理员重发
token 已过期401 / Close 4003 (expired)续期或申请新 key

建议客户端指数退避重连（1s → 2s → 4s … 上限 30s）；遇到 4003 不要无限重连。

10. 延迟说明

发帖 → trumpstruth.org RSS / Mastodon RSS30 – 80 秒（上游索引节奏）
上游 → 本服务轮询命中（10s 间隔，多源并发）0 – 10 秒
本服务 → WS 客户端< 50 ms

合计 30 – 90 秒。如果服务器能直连 truthsocial.com 一手 API（默认每 3 秒），上界可压到约 3 – 10 秒。
