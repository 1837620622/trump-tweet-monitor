📖 WebSocket 对接文档

首页 实时演示 管理后台

本服务通过 WebSocket 把特朗普 Truth Social 上的新帖推送到你的客户端 / AI 服务。下文所有示例已在生产中验证可复制即跑。

1. 端点 & 鉴权

WS 端点wss://<your-host>/ws （明文 IP/端口环境用 ws://<ip>:<port>/ws）
HTTP 基址https://<your-host>
鉴权方式 A（推荐，浏览器/简单 SDK）URL 参数：?token=ttws_live_xxx
鉴权方式 B（服务端调用）HTTP 头：Authorization: Bearer ttws_live_xxx
鉴权失败HTTP 401，连接断开
过期/吊销已建立的连接会在下次心跳时被踢出，close code = 4003
API Key 来自管理后台 /admin。创建时可选过期：1 小时 / 1 天 / 1 年 / 永久 / 自定义。

2. 连接生命周期

client                        server
  | --- HTTP Upgrade ----->     |
  | <--- 101 Switching  ---  |
  | <--- hello              ---|   服务器主动发的第一帧
  | --- subscribe (opt) --->   |
  | <--- subscribed         ---|
  | <--- tweet (real-time) ----|   每来一条新帖一帧
  | <--- ping (every 25s) -----|   WS 协议层 ping
  | ---  pong  ------------>   |   ws 库自动回 pong
  | --- ping (JSON, opt) -->   |   应用层心跳
  | <--- pong (JSON)         ---|

3. 客户端 → 服务端 消息

type字段说明
ping—应用层心跳；服务端回 pong
subscribesource: "all"|"rss"|"truth"过滤数据源（默认 all）
historylimit?: 1..200 (默认 20)请求最近 N 条历史

4. 服务端 → 客户端 消息

type关键字段说明
hellots, server_version, key连接建立第一帧；key 包含当前用的 id / prefix / expires_at
subscribedsource订阅成功确认
tweetdata: Tweet实时新帖
historydata: Tweet[]历史回放
pongts心跳应答
errormessage错误（不会自动断）

Tweet 数据结构

{
  "guid":        "https://truthsocial.com/@realDonaldTrump/posts/12345",
  "source":      "rss",           // 或 "truth"
  "title":       "原文纯文本",
  "contentHtml": "<p>原 HTML</p>",
  "link":        "https://truthsocial.com/...",
  "pubDate":     "2026-05-18T14:47:44.000Z",
  "author":      "@realDonaldTrump",
  "mediaUrl":    "https://.../image.jpg",   // 没图时 null
  "translated":  "中文翻译 (服务端预翻好)",
  "received_at": "2026-05-18T14:47:46.123Z",
  "raw":         { ... }                    // 调试用，AI 不要喂
}

给 AI 喂数据建议：把 title（英文原文）和 translated（中文）一起放进 prompt，再附 pubDate / source / link 作为上下文。

5. Python 客户端示例（粘贴即跑）

import asyncio, json, websockets

URL = "wss://YOUR-HOST/ws?token=ttws_live_XXXXXXXXXXXXXXXXX"

async def main():
    async for ws in websockets.connect(URL, ping_interval=20):
        try:
            await ws.send(json.dumps({"type": "subscribe", "source": "all"}))
            await ws.send(json.dumps({"type": "history", "limit": 20}))
            async for raw in ws:
                msg = json.loads(raw)
                if msg["type"] == "tweet":
                    t = msg["data"]
                    print(f"[{t['source']}] {t['pubDate']} | {t['title'][:80]}")
                    # 这里调用你的 AI / 写库 / 触发交易
                elif msg["type"] == "history":
                    print(f"got {len(msg['data'])} historical tweets")
        except websockets.ConnectionClosed:
            print("reconnecting…")
            continue

asyncio.run(main())

6. Node.js 客户端示例

const WebSocket = require('ws'); // npm i ws
const URL = 'wss://YOUR-HOST/ws?token=ttws_live_XXXXXXXXXXXXXXXXX';

function connect() {
  const ws = new WebSocket(URL);
  ws.on('open', () => {
    ws.send(JSON.stringify({ type: 'subscribe', source: 'all' }));
    ws.send(JSON.stringify({ type: 'history', limit: 20 }));
  });
  ws.on('message', raw => {
    const m = JSON.parse(raw);
    if (m.type === 'tweet') {
      console.log(`[${m.data.source}] ${m.data.title.slice(0, 80)}`);
      // -> 你的 AI 调用
    }
  });
  ws.on('close', () => setTimeout(connect, 1500));
  ws.on('error', e => console.error(e.message));
}
connect();

8. 错误码 & 重连建议

状况HTTP / WS Close说明
未带 tokenHTTP 401 (missing_token)客户端补 token 后重连
token 格式不对401 (bad_format)token 必须 ttws_live_ 开头
token 不存在401 (not_found)已被删除
token 已撤销401 / Close 4003 (revoked)联系管理员重新发
token 已过期401 / Close 4003 (expired)续期或申请新 key

建议客户端：指数退避重连（1s → 2s → 4s … 上限 30s）；遇到 4003 不要无限重连，先报警。

9. 延迟说明

这里说实话，给你管理预期：

链路典型耗时
Trump 在 Truth Social 发帖 → 出现在 trumpstruth.org RSS5 ~ 60 秒（其归档站轮询节奏）
服务端 RSS 轮询命中（默认 10s 间隔）0 ~ 10 秒
服务端 → WS 客户端< 50 ms（同机房）

合计 5 ~ 70 秒。如果服务器能直连 truthsocial.com 一手 API（默认每 3 秒），上界可压到约 3 ~ 10 秒。
